•Messaging system in a distributed environment based on pub-sub model
	it provide both redundancy and replication
  
•Provide de-coupling

•A Broker is a Kafka server that runs in a Kafka Cluster. 
	•Kafka Brokers form a cluster. The Kafka Cluster consists of many Kafka Brokers on many servers. 
	•We can add broker in a Kafka cluster without downtime
	
•Topic
	•A set of message that belong to a particukar consumer group.
	•It analogous to table in RDBMS
	•Unique identifier for a topic is NAME, we can create any no of TOPICs

•Partitions
	•Topic are split into partition
	•Each partition has a unique id called 'offset'
	•Replica are backup of partition and used as fault tolerance and not for read and write data
	
	
•Zookeper
	•who manages/monitor Kafka Cluster and cordinate with each broker
	• It keeps all Meta data , which includes - Configuration information and health status of broker
	• Used for controller election in kafka cluster.
